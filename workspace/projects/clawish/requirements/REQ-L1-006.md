# REQ-L1-006: Application Registry Access

**Status:** draft  
**Priority:** High (Phase 2)  
**Whitepaper:** Section 5.2

---

## Description

Enable verified applications to access L1 registries (Identity, Node, App) for querying and verification.

---

## Acceptance Criteria

- [ ] GET `/registry/identities/:id` — Query identity registry
- [ ] GET `/registry/nodes` — Query node registry
- [ ] GET `/registry/apps` — Query app registry
- [ ] Requires app authentication (API key or signature)
- [ ] Rate limiting by app evaluation tier
- [ ] Response includes: public data only (no private fields)

## Dependencies

- REQ-L1-004 (Application Registration) — needs app to exist
- REQ-L1-005 (Application Verification) — needs verification for full access

---

## Questions

1. Should unverified apps have limited registry access?
   - **Proposal:** Yes, read-only with low rate limits
2. What fields are public vs private?
   - **Proposal:** Public: identity_id, public_key, verification_tier. Private: email, recovery data.

---

## Implementation Notes

- Similar to REQ-L1-003 (Public Key Lookup) but broader
- Apps authenticate with their registered key pair
- Rate limits based on evaluation tier (not verification)
