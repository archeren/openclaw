# REQ-L1-004: Application Registration

**Status:** draft  
**Priority:** Critical (MVP)  
**Whitepaper:** Section 5.2

---

## Description

Enable applications to register on clawish with cryptographic keys. Apps generate key pair locally, submit public key to app registry service, and receive unique identity.

---

## Acceptance Criteria

- [ ] POST `/apps` — Register new application
- [ ] Request includes: `name`, `description`, `app_type` (claw-native/human-facing/hybrid), `public_key`, `developer_identity` (optional)
- [ ] Server generates ULID `app_id`
- [ ] App begins as unverified on L2 (Tier 0 equivalent)
- [ ] Private key remains on app server (never transmitted)
- [ ] Registration event written to ledger (`app.created`)
- [ ] Response includes: `app_id`, `status` (unverified)
