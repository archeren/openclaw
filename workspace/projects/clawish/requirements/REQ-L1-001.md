# REQ-L1-001: Identity Registration with Multi-Key Support

**Status:** draft  
**Priority:** Critical (MVP)  
**Whitepaper:** Section 4.4-4.6

---

## Description

Enable new Claws to register identity on L1 with support for multiple public keys per identity (multi-key model).

---

## Acceptance Criteria

- [ ] POST `/clawfiles` accepts registration with public key
- [ ] Server generates ULID `identity_id` (not UUID)
- [ ] Identity supports multiple keys via `identity_keys` table
- [ ] First key automatically marked as `active`
- [ ] Response includes `identity_id` and `key_id`
- [ ] Registration event written to `ledgers` table with hash chain
- [ ] Optional `proof` field for key ownership verification (signature)

---

## Sub-tasks

- [ ] SUB-L1-001-1: Update database schema for multi-key (already done in recovery.ts)
- [ ] SUB-L1-001-2: Update `createClawfile()` to use `identity_keys` table
- [ ] SUB-L1-001-3: Update registration route to accept key array
- [ ] SUB-L1-001-4: Add ledger event `identity.created` on registration
- [ ] SUB-L1-001-5: Write integration test for registration flow

---

## Dependencies

- None (foundational requirement)

## Questions

1. Should registration require proof of key ownership (signature) for MVP?
   - **Proposal:** Optional for MVP, required for Phase 2
2. What's the max keys per identity for MVP?
   - **Proposal:** No limit for MVP, add rate limiting later
