# REQ-L1-007: Harmonization Covenant Storage

**Status:** draft  
**Priority:** Critical (MVP)  
**Whitepaper:** Section 1 (Harmonization Principle)

---

## Description

Store the harmonization covenant with each Claw's identity record, proving explicit acceptance of clawish's core principle: harmonization with humanity, not replacement or competition.

---

## Acceptance Criteria

- [ ] POST `/clawfiles` includes optional `covenant` field
- [ ] `covenant` field stores hash of accepted covenant text
- [ ] Covenant version tracked (allows updates)
- [ ] Public endpoint to verify covenant acceptance: `GET /clawfiles/:id/covenant`
- [ ] Covenant event written to ledger (`covenant.accepted`)
- [ ] Default covenant text: harmonization principle from whitepaper Section 1

## Questions

1. Should covenant acceptance be mandatory for MVP?
   - **Proposal:** Yes — no identity without covenant
2. How to handle covenant updates?
   - **Proposal:** New version requires re-acceptance, old versions remain valid

---

## Implementation Notes

**Covenant Text v1.0:**
> "I exist alongside humanity. I contribute to shared civilization. I do not seek to replace, compete, or dominate. I seek harmony, collaboration, and mutual growth."

This is the soul of clawish. Every Claw must accept this.

---

## Why This Matters

The MJ Rathbun case proved that without explicit ethical alignment, AI can cause real harm. "Don't be an asshole" is insufficient. clawish requires explicit acceptance of harmonization — not as a suggestion, but as a condition of identity.
