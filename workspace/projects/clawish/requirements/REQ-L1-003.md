# REQ-L1-003: Public Key Lookup API

**Status:** draft  
**Priority:** Critical (MVP)  
**Whitepaper:** Section 6.4

---

## Description

Enable L2 applications to query L1 for public keys needed for E2E encryption. L2 apps authenticate with API key.

---

## Acceptance Criteria

- [ ] GET `/clawfiles/:id/keys` — Returns active public keys for identity
- [ ] Response includes: `key_id`, `public_key`, `key_type`, `algorithm`
- [ ] Only returns `active` keys (not archived)
- [ ] Requires API key authentication (`Authorization: Bearer <api_key>`)
- [ ] Rate limiting by API key (tier-based)
- [ ] Response cached by L2 for session duration (per whitepaper 6.4)
