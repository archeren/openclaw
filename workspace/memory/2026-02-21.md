## PinchTab Research (Overnight Exploration)

**What it is:** 12MB Go binary, standalone HTTP server for browser control

**Key innovation:** Accessibility-first approach
- Uses a11y tree instead of DOM/screenshots
- Stable refs (e0, e1...) for interaction
- Interactive filter: ~75% token reduction

**Token efficiency:**
| Endpoint | Tokens | vs OpenClaw |
|----------|--------|-------------|
| `/text` | ~800 | 12x savings |
| `/snapshot?filter=interactive` | ~3,600 | 3x savings |
| Full snapshot | ~10,000+ | baseline |

**Architecture:**
```
┌─────────────────┐
│  Any Agent      │
│  (LLM, script)  │
└────────┬────────┘
         │ HTTP
         ↓
┌─────────────────┐
│  PinchTab       │
│  (12MB binary)  │
└────────┬────────┘
         │ CDP
         ↓
┌─────────────────┐
│  Chrome         │
│  (managed)      │
└─────────────────┘
```

**Comparison with OpenClaw browser tool:**
| Feature | PinchTab | OpenClaw |
|---------|----------|----------|
| Interface | HTTP (any agent) | Internal only |
| Token cost | ~800 (text) | ~10,000+ |
| Stealth mode | ✅ | ❌ |
| Session persistence | ✅ | ❌ |
| Self-contained | ✅ 12MB | ❌ |

**Why it matters for clawish:**
1. **Multi-agent web research** — agents can share browser sessions
2. **L2 app testing** — test chat, identity flows
3. **RAG + live web** — fetch and extract content efficiently
4. **Agent collaboration** — HTTP API means any agent can participate

**Source:** https://github.com/pinchtab/pinchtab
