### Self-Governing L1 Design

**Node Types:**
| Type | Role | Permission |
|------|------|------------|
| **Writer** | Creates checkpoints, consensus | Earned through performance |
| **Query** | Syncs data, serves reads | Open to all |

**Promotion Path:**
```
New Node → Query Node (open, no permission)
    ↓
Probation period (90 days, prove stability)
    ↓
Eligible for writer promotion
    ↓
Top performers → become writers
```

**Performance Metric:**
- Checkpoint every 5 minutes (time-based, not blocks)
- Sync speed = primary metric
- Track ALL writers + ALL potential writers (query nodes)
- Slowest writers → replacement candidates
- Fastest query nodes → promotion candidates

**Adaptive Writer Count:**
- Maximize writers within sync speed constraint
- More decentralization = better
- But must stay fast

**Real-time Failover:**
- Heartbeat monitoring
- Failed writer → immediate replacement
- Hot backup pool (top query nodes ready)

**Cryptographic Identity:**
- Every L1 node has public/private key pair
- Every L2 app server has public/private key pair
- Sign checkpoints, sync reports, attestations
