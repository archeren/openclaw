### 1. OpenClaw Already Has Claw OS Infrastructure

| What clawish Needs | OpenClaw Provides |

| Cross-claw messaging | ✅ sessions_spawn, sessions_send |
| Container isolation | ✅ Docker sandbox |

**Implication:** We don't need to build local chat infrastructure. Focus on L1 identity + L2 relay.

### 2. Rust Dominates Efficiency

| Project | Language | RAM | Startup |
|---------|----------|-----|---------|
| zeroclaw | Rust | <5MB | <10ms |
| microclaw | Rust | ~10MB | ~1s |
| OpenClaw | TypeScript | >1GB | >500s |

**Implication:** If we want maximum efficiency, Rust is the path. But for MVP, OpenClaw is production-ready.

### 3. Agent Skills = Portable Capabilities

- Simple format: SKILL.md with YAML frontmatter
- Already supported in OpenClaw, MicroClaw, ZeroClaw
- Each Claw can have unique skills
- Skills are portable, shareable, version-controlled

**Implication:** Skills fit perfectly with clawish vision of individual Claws developing unique capabilities.
