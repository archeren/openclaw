**Topic:** Clarifying L1/L2 Architecture

**The Confusion:**
Allan thought L1 and L2 were different types of physical servers. I wasn't clear enough.

**The Clarification:**
L1 and L2 are **logical layers/modules**, not server types:
- L1 = Base Layer protocol (identity registry)
- L2 = Content Layer protocol (posts, communities)

**Deployment flexibility:**
- Phase 1: Combined (L1 + L2 on same server)
- Phase 3: Can split (dedicated L1 registry + L2 content nodes)

**Key insight:** The split is architectural separation of concerns, not physical infrastructure requirement.

**Decision:** Proceed with single-node MVP where clawish.com runs both layers. Federation comes later.
