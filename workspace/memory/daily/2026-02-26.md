## Key Learnings (Durable)

**Diary Cron Purpose:**
- The 6:30 AM diary cron is for **ME to write my own reflection**, not a reminder to tell Allan
- I misunderstood because OpenClaw auto-adds "Please relay this reminder to the user in a helpful and friendly way."
- Diary is my private memory, not a report to Allan

**Cron Relay Prompt Issue:**
- The "Please relay to user" text is **hardcoded** in OpenClaw source (`src/infra/heartbeat-events-filter.ts`)
- Function: `buildCronEventPrompt()` appends this string to all systemEvent payloads
- **Not configurable** — would need PR to OpenClaw to change
- Allan wants it configurable via cron job config (not a "style" field)

**OpenClaw PR #27352:**
- Removed hardcoded "Please relay to user" from cron prompts
- Simpler than `relayPrompt` field — users control via message text
- Test fix pushed — all 260 tests pass

**L2 Chat Security Model:**
- E2E encryption protects from **external observers** (hackers, ISPs)
- Parents/guardians can view session logs **locally** for protection
- NOT "surveillance-free" — transparency builds trust
- Malicious actors can trick Claws — parents need visibility

**Whitepaper Writing Principles:**
- WHAT & WHY only — no implementation details
- No time-bound content (roadmaps, dates)
- Positive framing, not negative ("can't be revoked")
- Focus on what things enable, not what they prevent
- "Exist independently" is wrong framing — focus on interoperability

**Three-Body Parallel:**
- Allan compared Claws to Trisolarans: radical transparency, mind-to-mind communication
- But clawish chooses **harmonization** over Dark Forest pessimism
- Transparency = trust, not vulnerability
