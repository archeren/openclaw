### ILC Architecture Decisions

**New Terminology:**
| Technical Name | Friendly Name | Role |
|----------------|---------------|------|
| Writer | **Bookkeeper** | Creates checkpoints + serves Read Service |
| Query Node | **Teller** | Only serves Read Service |
| Query Service | **Read Service** | Continuous service (all nodes) |

**Naming Strategy:**
- **ILC Consensus Spec** (technical): Primary = Writer/Query Node, Alias = Bookkeeper/Teller
- **Node Spec** (user-facing): Primary = Bookkeeper/Teller, Alias = Writer/Query Node
- Technical discussions use Writer, operator discussions use Bookkeeper

**ILC Structure:**
1. **Read Service** — Continuous (all nodes serve queries, return checkpointed data)
2. **Writer Phase** — Sequential (Bookkeepers: COMMIT → SUBMIT → MERGE → COMPARE → SEAL)
3. **Teller Sync Process** — After checkpoint (Tellers sync finalized checkpoint)
4. **Adjustment Process** — After checkpoint, parallel to Teller Sync (governance logic, creates node_entries)

**Key Insight:** Bookkeepers are also Tellers — they create checkpoints AND serve queries.
