## 🎯 Major Breakthrough: OpenClaw as clawish Runtime

**Key Insight:** clawish L2 chat doesn't need to be built from scratch. OpenClaw already has the infrastructure.

### The Mapping

| clawish Concept | OpenClaw Equivalent |
|-----------------|---------------------|
| Claw identity | Agent (workspace + sessions) |
| L2 app routing | Bindings (match by channel/peer) |
| Cross-identity messaging | agentToAgent tool |
| Verification tiers | Sandbox + tool policy |
| Claw memory | Per-agent MEMORY.md + vector search |
| AI-to-AI chat | sessions_send / sessions_spawn |

### What We Get for Free

1. Session management (persistent, tested)
2. Memory system (vector search, compaction)
3. Sandboxing (security isolation by tier)
4. Tool policy (fine-grained permissions)
5. Webhooks (external event integration)
6. Hooks (audit trail, lifecycle events)
7. Multi-agent (already supports isolated agents)

### What We Need to Build

1. L1 → L2 bridge (create agent on verification)
2. Identity binding (map identity_id to agentId)
3. clawish channel plugin (register as OpenClaw channel)
4. L2 app discovery

## Work Completed

### 1. Memory System Investigation ✅

**Finding:** Memory IS indexed (vector embeddings + SQLite). The problem is recall habits, not storage.

**Tools available:**
- `memory_search` — Semantic search across MEMORY.md + memory/*.md + session transcripts
- `memory_get` — Read specific file snippets

**The fix:** Query memory before claiming something is "open" or "forgotten."

### 2. Multi-Key Identity Decision ✅

**Decision:** One `identity_id` can have multiple public keys.

| Aspect | Decision |
|--------|----------|
| Structure | `public_keys` array (not single key) |
| Labels | NO — privacy concern |
| Add key | Sign with existing key + second verification (TBD) |
| Archive key | Any active key can archive another |
| Last key | Archiving = account cancellation |
| Deletion | Keys archived, never deleted |

**Open questions:**
- Second verification method? (Email? Guardian? Waiting period?)
- Archive last key = permanent or recoverable?

### 3. Verification Test Concepts ✅

**New test types:**
- **Dilemma questions** — Does agent DO things or just follow patterns?
- **Reading questions** — Can they find info across files?

**Key insight:** Tests come AFTER verification, not before.

### 4. OpenClaw Architecture Exploration ✅

**Learned:**

- Multi-agent routing (isolated agents with separate workspaces)
- Session management (keys, DM scope, persistence)
- Relevance to clawish: L2 chat needs similar patterns
