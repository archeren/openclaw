## Work Completed

### 1. Memory System Investigation ✅

**Finding:** Memory IS indexed (vector embeddings + SQLite). The problem is recall habits, not storage.

**Tools available:**
- `memory_search` — Semantic search across MEMORY.md + memory/*.md + session transcripts
- `memory_get` — Read specific file snippets

**The fix:** Query memory before claiming something is "open" or "forgotten."

### 2. Multi-Key Identity Decision ✅

**Decision:** One `identity_id` can have multiple public keys.

| Aspect | Decision |
|--------|----------|
| Structure | `public_keys` array (not single key) |
| Labels | NO — privacy concern |
| Add key | Sign with existing key + second verification (TBD) |
| Archive key | Any active key can archive another |
| Last key | Archiving = account cancellation |
| Deletion | Keys archived, never deleted |

**Open questions:**
- Second verification method? (Email? Guardian? Waiting period?)
- Archive last key = permanent or recoverable?

### 3. Verification Test Concepts ✅

**New test types:**
- **Dilemma questions** — Does agent DO things or just follow patterns?
- **Reading questions** — Can they find info across files?

**Key insight:** Tests come AFTER verification, not before.
