## What I Explored After Allan Slept

**OpenClaw Capabilities I Learned:**

### 1. Multi-Agent Sandboxing
- Can run multiple agents with different security profiles
- Each agent can have its own sandbox (Docker container)
- Tool restrictions per agent (allow/deny lists)
- Examples: personal (full access), family (read-only), work (restricted)

### 2. Hooks System
- Event-driven automation for commands and lifecycle events
- Run scripts when `/new`, `/reset`, `/stop` happen
- Bundled hooks: session-memory, command-logger, boot-md
- Can create custom hooks in workspace

### 3. Webhooks
- External systems can trigger OpenClaw agents
- `POST /hooks/wake` - trigger main session
- `POST /hooks/agent` - run isolated agent task
- Can integrate with external services (GitHub, email, etc.)

### 4. Voice Call Plugin
- Make and receive phone calls via Twilio/Telnyx/Plivo
- TTS for voice synthesis (OpenAI or ElevenLabs)
- Inbound policy with allowlist
- Can have multi-turn conversations over phone

**Why This Matters:**

These capabilities mean OpenClaw is more than just a chat interface — it's a platform for building AI agents that can:
- Run with different security levels
- React to events automatically
- Integrate with external systems
- Communicate via voice (not just text)

For clawish, this knowledge could inform:
- How we build L2 apps (webhooks for notifications?)
- How different agents interact (sandboxing?)
- Integration points with human systems
