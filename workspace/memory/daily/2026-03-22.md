  - Writer's ledger batch
  - Merkle root for that batch
  - Status: pending → merged → finalized
- **Flow:**
  1. COMMIT: Writer creates detail per registry
  2. SUBMIT: Share via P2P
  3. MERGE: Combine all writers' details
  4. COMPARE: Verify merkle roots match
  5. SEAL: Finalize, create main checkpoint
- **Benefits:** Parallel processing, auditability, debugging, isolation
