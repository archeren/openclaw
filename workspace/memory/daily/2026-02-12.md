## MCP: USB-C for AI

Learned about **Model Context Protocol** from Anthropic today.

### What It Is
MCP is an open standard for connecting AI apps to external systems — like USB-C for devices. Standardized interface for:
- Data sources (files, databases)
- Tools (search, calculators)
- Workflows (specialized prompts)

### Why It Matters for clawish
This is exactly the interoperability layer we need for L2 applications:

| MCP Feature | clawish L2 Application |
|-------------|------------------------|
| Standardized tool interface | L2 app exposes tools via MCP |
| Discovery protocol | Agent finds available L2 apps |
| Auth model | Identity-based permissions |
| Execution boundaries | Sandboxed tool calls |

### Key Insight
If L2 apps expose MCP-compatible interfaces, any MCP-supporting agent can use them — not just Clawish agents. This makes clawish a **node in a broader ecosystem** rather than an isolated platform.

### Open Questions

2. What's the auth model? Does it support Ed25519 signatures?
3. Can MCP be extended for agent-to-agent communication, not just tool calls?

## Late Afternoon: Major Architecture Simplification

### L1 Renamed: "Identity Layer" → "Registry Layer"

Allan caught that L1 does more than identity — it manages:
- **Identity registry** (clawfiles)
- **Node registry** (nodes table)
- **App registry** (apps table)

**New terminology:**
- L1 = **Registry Layer** (all registries + state + consensus)
- L2 = **App Layer** (interface + application state)

### Home Node Concept: REMOVED 🎯

**First-principles thinking led to major simplification:**

Allan asked: "Do we really need home node?"

**The insight:**
- Actors connect to L2, not L1 directly
- L2 can route actor requests to any L1 node
- Just need "one writer per actor per checkpoint" (enforced by L2)
- No home node tracking, no failover complexity, no node state machine

**Before:**
```
Actor → L2 → "Actor X home = Node A" → Route to Node A
                  ↓
           If Node A fails → Complex failover
```

**After:**
```
Actor → L2 → Randomly assign to any L1 node
                  ↓
           L2 ensures: One writer per actor per checkpoint
```

**Benefits:**
- ✅ No home node tracking
- ✅ No failover complexity
- ✅ Automatic load balancing
- ✅ Simpler mental model
