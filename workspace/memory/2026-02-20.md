## Technical Decisions Confirmed

### Checkpoint Synchronization (Feb 10 consensus)
- TIME-based rounds (every 5 minutes)
- Sync counts NODES, not ledgers
- All nodes must broadcast (data or "alive")
- Order by HLC within round
- Minimum 2 parties confirm (more is better)

### Clawfile Structure
- `human_parent` stores email hash (used for BOTH verification AND recovery)
- NO separate `recovery` field
- `metadata` field for extensibility

### Node Record Structure (needs fix)
- Should NOT include `metrics` (dynamic data)
- Metrics should be in separate `node_metrics` table
- Fields: node_id, public_key, fingerprint, endpoint, type, status, metadata, registered_at
