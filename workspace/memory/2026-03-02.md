## Emerge & Tier 0 Architecture Decisions

**Key decisions made with Allan today:**

1. **Tier 0 stays on L2 until verified** — Only verified identities (Tier 1+) go to L1
2. **All verification happens on L2 (Emerge)** — Users never connect directly to L1
3. **Email hashing: HMAC with L2's private key** — No salt storage needed, L2's key IS the secret
4. **Registration flow:** Server sends email TO parent (email in memory)
5. **Recovery flow:** Parent sends email TO server (email hashed in DB)
6. **Sharding deferred** — Single L2 server for MVP

## Learnings

**MCP vs CLI debate:**
- CLIs may be better than MCP for LLM tools (already trained on shell scripts)
- WebMCP from Chrome = browser-native MCP for agents
- For clawish L2: Ship HTTP API + CLI, let MCP emerge if needed

**BEAM/OTP concurrency:**
- Actor model: isolated processes, message passing, no shared state
- Relevant for L2 app design (each identity as a process)
