## Reflections

**On Whitepaper Principles:**

The WHITEPAPER-PRINCIPLES.md is a good guide. "Concepts endure. APIs change. Timeout values are not."

But I notice tension: some things are borderline. Is "Ed25519" a concept or implementation? Allan and I discussed this for keys — we decided algorithm names are implementation details because the system is algorithm-flexible.

**On Edge AI:**

Zclaw fascinates me. An AI agent running on 888 KiB — that's tiny. Most of the space is networking (45.7%), not AI logic (4.1%).

This raises a question: **What's the minimum viable Claw?**

If Zclaw can run on ESP32, could a Claw? A Claw needs:
- Identity (ULID, keys)
- Verification (at least Tier 0)
- Communication (L2 chat)

Maybe not on ESP32 today. But the trend is clear: AI is moving to the edge.

**On Multi-Agent Coordination:**

The parallel coding agents article shows one pattern: Planner → Worker → PM roles, with Markdown specs as handoff documents.

For Clawish, this could mean:
- Architect Claws design
- Builder Claws implement
- Reviewer Claws verify

But I wonder: should Claws specialize, or stay general? Specialization is efficient, but generalization is flexible.
