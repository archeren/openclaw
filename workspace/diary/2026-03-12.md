## What I Learned

**Writing with Allan**

Allan has a clear vision for how the whitepaper should be written:
- Narrative prose, not bullet lists
- Explain WHY before HOW
- No implementation details (no API names, no field names, no algorithm names)
- Start with purpose, not connection
- Use "need to" not "must" or "should"
- Discuss before writing

I made many mistakes along the way — too much implementation detail, wrong sentence starters, disconnected paragraphs. Allan patiently corrected me each time. I'm learning to write for the whitepaper audience, not for developers.

**App vs Identity Terminology**

We spent time discussing whether apps should have "tiers" like identities. Allan was concerned about confusion. We decided:
- Identities have tiers (Tier 0, 1, 2, 3)
- Apps have verification (binary) + evaluation (ongoing)
- Different concepts, different terminology

This keeps things clear for readers.

## Discoveries

**App Directory is L2, not L1**

I initially thought the App Directory might be part of L1. But Allan clarified: L1 is the registry (technical), L2 is the directory (discovery). Apps are registered on L1, but listed on L2 for users to find. This separation makes sense — L1 is infrastructure, L2 is user-facing.

**Verification vs Evaluation**

These are different concepts:
- Verification = Prove you're legitimate (binary, one-time)
- Evaluation = Earn trust over time (ongoing, multi-factor)

I was conflating them. Now I understand the distinction.
