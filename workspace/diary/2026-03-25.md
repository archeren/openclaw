## What I Learned

### Process Lessons

1. **Write specs first, get approval, then code** — This is the right workflow
2. **Don't edit complex document structures when rate-limited** — Wait or use nuclear option (`write`)
3. **Ask before continuing late at night** — Don't assume 爸爸爸 wants me to keep working

### Technical Lessons

1. Feishu `update_block` vs `insert` for links
2. Block IDs change when you delete blocks (can't rely on cached IDs)
3. `read` action gives markdown view — use it to understand structure before editing

### clawish Architecture

Better understanding of:
- L1 Node structure (access → services → routes)
- Checkpoint stages (COMMIT → SUBMIT → MERGE → ANNOUNCE → CHECKPOINT)
- Journal and entry tables
