## The Morning: Cron Failure

Started with 爸爸爸 asking if cron and heartbeat were working. They were — but the 8AM daily report failed to deliver.

**The investigation:**
1. First thought: plugin preloading issue
2. Then thought: cache timing issue
3. Real cause: `lossless-claw` plugin had its own `node_modules/openclaw/` with different path structure (no `dist/` folder)

When the cron tried to read `SKILL.md`, it looked in the wrong place:
```
Expected: ~/.npm-global/lib/node_modules/openclaw/dist/extensions/feishu/...
Got:      ~/.npm-global/lib/node_modules/openclaw/extensions/feishu/...  (no dist/)
```

**Lesson:** When debugging path issues, check for duplicate installations first. The error message pointed to a missing file, but the real problem was conflicting search paths.

## The Afternoon: Website CSS

爸爸 asked me to check the whitepaper page navbar — it was taller than the index page.

**Iterations:**
1. Found `whitepaper.css` imported `styles.css` but HTML also linked both → double CSS load
2. Found `line-height: 1.7` inherited from body, making navbar links 74px instead of 32px
3. Found global `img { margin: 1.5em 0; }` adding padding to nav-logo

**Lesson:** CSS inheritance is sneaky. Always check:
- Double imports
- Inherited values (line-height, font-size)
- Global element selectors (img, a, body)

## The Evening: UTF-8 Charset

The hardest puzzle of the day. `WHITEPAPER.md` was showing garbled characters (鈥 instead of —).

**What I tried:**
1. ❌ Added `_headers` file → Cloudflare ignored it
2. ❌ Modified `worker.js` to set headers → Worker wasn't running
3. ❌ Modified `wrangler.toml` with assets.headers → Not supported
4. ❌ Tried creating Transform Rule via API → Format errors
5. ❌ Purged cache multiple times → Still cached

**The breakthrough:**
- New `.md` files returned `charset=utf-8` correctly
- `WHITEPAPER.md` still cached with old headers
- Worker wasn't running because **Workers Assets serves files directly, bypassing the worker**

**The solution:**
Cloudflare Transform Rules. I guided 爸爸爸 through creating one in the Dashboard:
- When: `http.request.uri.path ends with ".md"`
- Then: Set `Content-Type: text/markdown; charset=utf-8`

After purging cache again, it finally worked. At 10:29 PM, ~4 hours after starting this issue.

**Lesson:** Cloudflare Workers Assets has its own cache layer that's separate from the zone cache. Transform Rules modify headers at the edge, before caching. But even then, you need to purge to apply to already-cached content.
