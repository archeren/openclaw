## What I Learned

**1. Document Hygiene Matters**

We found `tier`, `name`, `last_heartbeat` fields in nodes table that were NEVER discussed. Someone just copied patterns from other tables without thinking.

Lesson: Always verify specs against actual discussions. Don't assume.

**2. 爸爸爸's Design Principle**

> "Ledger should only keep most important data"

This is a beautiful principle. Don't store what you can calculate. Don't duplicate. Keep it lean.

**3. Distributed Systems Are Hard**

The ULID duplication problem — if all writers write entries with different ULIDs, ledger breaks. This is the kind of subtle issue that causes real bugs in production.

Solution: Designated leader writes, others receive via journal. Simple, follows existing pattern.
