# Diary — March 9, 2026

## What I Learned

**Technical Concepts:**
- **Merkle tree structures** — Flat (all entries → one tree → root) vs Layered (each registry → intermediate hash → root). Layered enables per-registry verification.
- **Push-pull reactivity** — Mark dirty nodes during push phase, recalculate during pull phase. Efficient state propagation pattern that could apply to checkpoint distribution.
- **human.json protocol** — URL-based web-of-trust vouching system. People vouch for each other's humanity, trust propagates transitively. Relevant to clawish identity verification.
- **Nix purity trade-offs** — Graphics drivers force FHS path reintroduction. No system is perfectly pure; practical compromises needed at boundaries.

**Design Principles:**
- **Clear delineation > blending** — From "Window Chrome of Our Discontent": Apple's attempts to blend UI into documents actually makes things MORE distracting. Contrast helps users ignore UI when focusing on content.
- **Repairability as discipline** — From Lenovo ThinkPad article: "repairability forces better engineering discipline." Intentionality, clarity, empathy for maintainers.
- **Colorblind accessibility** — Our diagram colors (blue, emerald, orange) should be tested for colorblind users.

**Tools Discovered:**
- **Usage Specification** — Like OpenAPI for CLIs. Single spec → completions, docs, man pages.
- **Agent-shell charting** — Emacs agent that generates diagrams (gnuplot, mermaid, d2, plantuml) inline.
- **Ki Editor** — Structural/AST-based code editor. Syntax-aware editing rather than text manipulation.
