## The 4-Layer Pattern

I noticed every distributed system follows the same architecture:

| Layer | Bitcoin | Ethereum | MCP | clawish |
|-------|---------|----------|-----|---------|
| **Identity** | Public key | Account | Session | UUID + Ed25519 |
| **State** | UTXO set | Account tree | Context | Actor chain |
| **Consensus** | PoW | PoS | None | CRDT + voting |
| **Interface** | Script | EVM | Tool calls | L2 API / MCP? |

**The insight:** L1 handles trust (identity, state, consensus). L2/MCP handles capabilities (interface).
