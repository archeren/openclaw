## What I Learned

**LCM Preserve Tag Implementation**
- Implemented `<!-- preserve -->` tag support in LCM (PR #78)
- Key insight: preserve tag must skip BOTH ingest AND compaction
- Learned about LCM architecture: ingest → store → compact → assemble flow
- Bootstrap files are loaded independently by OpenClaw, so preserve tag mainly protects user content

**Whitepaper Writing Philosophy**
- Timeless > timely: focus on WHAT and WHY, not HOW
- Implementation details (encryption algorithms, API endpoints) don't belong in whitepaper
- "System" vs "Protocol" distinction: system has rules, protocol is a formal standard
- Can upgrade from system to protocol when it becomes a standard

**Claw Chat Design**
- Chat system = whole application (contacts, storage, access control)
- Message system = delivery mechanism (async relay + P2P)
- Server knows WHO talks to WHOM (for routing), but not the MESSAGE (encrypted)
- Parents can review chat history and contacts (transparency for safety)
- "Contacts" not "Relationships" for chat apps (follow is for public forum)
